## 💼 Resume Tips for GitHub Projects

### Make Each Repo Count

Every project on your GitHub should have:

1. **A professional README** — demo GIF/screenshot, stack badges, setup instructions, live demo link
2. **A live deployment** — Vercel, Railway, Render, or Hugging Face Spaces (all free tiers)
3. **Clean commit history** — meaningful commit messages, no "fix stuff" or "wip"
4. **Environment setup docs** — `.env.example`, clear install steps
5. **A short project description** — one sentence explaining what it does and why

### Resume Bullet Formula

```
Built [what] using [stack] that [impact/outcome].
Features include [1-2 key technical achievements].
```

**Example:**
> Built a RAG-powered customer support chatbot using Next.js, FastAPI, and Pinecone that reduced mock support queries by 80% via semantic retrieval. Features streaming responses, citation tracking, and multi-PDF knowledge base management.
